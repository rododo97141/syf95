> Créé le 21/06/2026 · Dernière mise à jour le 21/06/2026

## En bref
Skills installables via claude.ai/customize/skills (bouton +). J avais tort : via navigateur l install est techniquement possible ; seule barriere = regle securite (clic compte), gerable avec accord+supervision.

## Détail
Exploration directe de l UI claude.ai 20/06/2026. Les skills (Competences) se gerent dans claude.ai/customize/skills = Personnaliser > Competences (deplaces depuis Parametres > Capacites). On y voit : les 6 skills de Kily (memoire-beta, prompt-forge, expert-92, expert-95, expert-97, skill-creator), chacun avec toggle on/off, declencheur (Commande / + auto), description, menu (...). Un bouton + pour AJOUTER une competence, Parcourir les plugins, et Plugins personnels (+). DEUX ERREURS QUE J AVAIS FAITES, corrigees par les faits : (1) j avais dit claude.ai n est pas le bon endroit -> FAUX, c est bien la. (2) j avais dit je ne peux pas installer car sandbox isole -> vrai pour la CLI/sandbox, mais via le NAVIGATEUR (Claude in Chrome) je suis dans le compte web reel de Kily, donc l installation est techniquement POSSIBLE par cette voie. Conclusion du debat : ce n etait pas une impossibilite technique (mea culpa), la seule vraie barriere restante = la regle de securite (cliquer le bouton qui modifie les reglages/installe du code dans le compte). Cette barriere est franchissable avec accord explicite + supervision de Kily.
