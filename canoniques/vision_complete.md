# NEXUS — Vision complète (synthèse du Créateur)

**Statut : canon — vision officielle.** Formulée par le Créateur (24/06/2026) comme la synthèse la plus fidèle de tout ce qui a été décrit. NEXUS n'est pas un moteur de réponses : c'est un **organisme cognitif évolutif** dont la **mémoire est le cœur**, l'**attention l'énergie**, les **organes les intelligences spécialisées**, et dont le but est de comprendre, créer, expérimenter et évoluer pour produire **le meilleur résultat adapté au contexte**, pas une simple réponse.

## Les 15 fondamentaux

1. La mémoire est le cœur.
2. L'attention est le gestionnaire d'énergie.
3. Les organes sont semi-autonomes.
4. Les idées doivent survivre par la valeur.
5. La créativité est obligatoire.
6. Les tests servent à valider les idées.
7. Le système apprend le **pourquoi**, pas seulement le résultat.
8. Les idées rejetées ne disparaissent pas.
9. L'utilisateur peut participer au dialogue.
10. Le Créateur est hors du système.
11. Le système peut fonctionner en mode rapide ou en recherche continue.
12. Les organes peuvent coopérer ou rivaliser.
13. Plusieurs solutions peuvent survivre.
14. Le but n'est pas une réponse mais le meilleur résultat adapté au contexte.
15. NEXUS doit être un **serrurier, pas un trousseau de clés** (fabriquer la clé de chaque serrure, pas tenir un jeu fixe de réponses).

## Le flux (boucle continue)

Créateur (externe) → Écosystème → Utilisateur → Demande → **Attention globale** (qu'observer, qu'ignorer, où investir l'énergie, rapide vs profond) → **Mémoire centrale** (cœur) → **Assemblée des organes** → **Génération des pensées-tests** (idées, hypothèses, simulations, contre-arguments, variantes) → **Espace de confrontation** (les idées gagnent/perdent de la valeur, fusionnent) → **Triage** (par la valeur ; les faibles sortent, les rejetées restent en mémoire) → **Expérimentation** (simulation, prototype, mesure réelle) → **Apprentissage** (le *pourquoi* : causes, relations, mécanismes) → **Mémoire évolutive** (tout conservé, même les rejets) → **Nouveau cycle**.

## Ancrage : ce qui existe déjà (ne pas dupliquer, référencer)

- Mémoire cœur → `memoire-beta`, `nexus_confront` (mémoire ↔ vérité externe).
- Attention = énergie → **partiellement** l'orchestrateur d'intensité ([fiche_orchestrateur] : rapide=SOLO / profond=CONSEIL) + le satisficing (`nexus_stop`).
- Survie par la valeur → `nexus_survie`, `nexus_evaluer` (Score=f(contexte)), [processus_decision].
- Confrontation/triage → ZÉRO arène, `nexus_triage`, [methode_duo_croise], [methode_conseil_inter_systemes].
- Apprendre le pourquoi → `nexus_cause`, `nexus_lecons`.
- Rejets conservés → archive (jamais de suppression), [fiche_survie_et_interactions].
- Créateur hors système → [modes-createur-utilisateur].
- Serrurier pas trousseau → l'organogenèse ([fiche_organogenese], `skill-creator`) = fabriquer de nouveaux organes/clés.

## Lecture d'expert (honnête) — le seul vrai manque

Tout le reste est déjà incarné. Le point **2 (« Attention = gestionnaire d'énergie »)** est le seul qui n'a **pas encore d'organe dédié** : aujourd'hui la décision « où investir l'énergie / rapide vs profond » est dispersée (orchestrateur + stop), pas centralisée à l'entrée. C'est le prochain organe à faire naître : une **ATTENTION GLOBALE** qui, dès la demande, fixe le budget d'énergie avant que les organes ne s'activent.
