# Dossier `modeles/` — expert-95

## Rôle

Ce dossier contient des **gabarits / templates réutilisables** : des **livrables
types** que le skill peut **instancier** (remplir, adapter) selon le cas d'usage.

Un modèle fournit la **structure** d'un livrable — les rubriques, l'ordre, le ton
attendu — pour produire vite et de façon **cohérente**, sans repartir d'une page
blanche.

## Distinction avec les autres dossiers

| Dossier          | Contenu                                                        |
| ---------------- | ------------------------------------------------------------- |
| `references/`    | **doctrine** du skill (méthode, commandes, orchestration).    |
| `connaissances/` | **savoirs** propres à un cas d'usage (ex. SYFIR).             |
| `procedures/`    | **modes opératoires** réutilisables (le « comment faire »).   |
| **`modeles/`**   | **gabarits de livrables** (le « quoi remplir »).             |

## Exemples de modèles possibles

- Gabarit de **bilan final** (cf. format du noyau `SKILL.md`).
- Gabarit d'**analyse** (*En bref / Points clés / Signaux et incohérences /
  Recommandations*).
- Gabarit de **fiche d'apprentissage** (*L'essentiel / Vocabulaire / Meilleures
  pratiques datées / Pièges / Chiffres / Sources*).
- Gabarit d'**entrée de journal** (*Date / Mission / Ce qui a fonctionné / Ce qui
  a échoué / Amélioration proposée*).
- Autres livrables types selon les domaines (plan de projet, brief, contrat-type
  générique, etc.).

> Ces modèles seront ajoutés au besoin. Plusieurs structures sont déjà décrites
> dans `references/` ; ce dossier les matérialise en **templates instanciables**.

## Principe d'usage

- **Lecture / instanciation** d'un modèle existant = **libre**.
- **Création / modification** d'un modèle = écriture durable → **sous
  autorisation** (`mémorise 95`).
- **Aucune donnée sensible** dans les gabarits : ils restent **génériques** et
  réutilisables.
