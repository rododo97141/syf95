# Dossier `procedures/` — expert-95

## Rôle

Ce dossier contient les **procédures opérationnelles réutilisables** du skill :
des **méthodes éprouvées**, **checklists** et **workflows** que le système
applique de façon répétable, quel que soit le cas d'usage.

Une procédure décrit **comment faire** une opération récurrente, étape par étape,
de manière à garantir la **constance** et la **qualité** des traitements.

## Distinction avec les autres dossiers

| Dossier          | Contenu                                                        |
| ---------------- | ------------------------------------------------------------- |
| `references/`    | **doctrine** du skill (méthode, commandes, orchestration).    |
| `connaissances/` | **savoirs** propres à un cas d'usage (ex. SYFIR).             |
| **`procedures/`**| **modes opératoires** réutilisables (le « comment faire »).   |
| `modeles/`       | **gabarits / templates** de livrables (le « quoi remplir »).  |

## Contenu prévu

- `handshake.md` — procédure d'activation du skill (`active 95`).
- *(d'autres procédures s'ajouteront au besoin : analyse, apprentissage,
  capitalisation, etc.)*

## Principe d'usage

- **Lecture / application** d'une procédure existante = **libre**.
- **Création / modification** d'une procédure = écriture durable → **sous
  autorisation** (`mémorise 95`).
- Avant d'en créer une nouvelle : **vérifier qu'elle n'existe pas déjà**
  (Handshake) pour éviter doublons et contradictions.
